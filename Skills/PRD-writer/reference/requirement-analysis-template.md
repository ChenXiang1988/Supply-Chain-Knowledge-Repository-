## 14. 填写示例（到货签收）

> 下面是一个填写示例，用来说明“用户是谁、用户做什么、期望得到什么结果”应该怎么写。

### 14.1 业务场景示例

- 场景名称：到货签收
- 用户是谁：仓库收货员
- 用户做什么：在 WMS 中打开到货单，按箱扫描实物，核对 SKU、箱数和件数，录入实收数量及破损、短少等异常情况
- 期望得到什么结果：系统生成签收记录，到货单状态更新为“已签收”，并触发后续上架或入库流程；异常信息同步给相关责任人

### 14.2 流程说明示例

1. 仓库收货员收到承运方送达的货物
2. 收货员在系统中找到对应到货单
3. 系统按单展示应收明细
4. 收货员完成扫描、核对和异常录入
5. 用户提交签收后，系统保存签收结果并更新单据状态
