## 2. 项目背景与问题分析
### 2.1 项目背景
目前公司国内仓仓储业务全程依赖第三方商用WMS系统支撑，覆盖收货、库内管理、订单履约、盘点、出库等核心场景。
随着跨境物流业务规模扩张，国内仓备货量、订单履约量、退换货单量逐年递增，业务场景日趋复杂。第三方WMS的标准化能力、定制化能力、数据自主权、成本可控性已无法匹配业务发展需求。

为实现仓储业务自主可控、流程精细化管控、数据全域打通、运营成本优化，同时适配跨境业务多渠道、多品类、高时效的履约特点，公司启动**国内仓第三方WMS全面切换自研WMS系统项目**，彻底替代现有第三方系统，搭建专属化、可迭代、高适配的国内仓储管理体系。

### 2.2 现有第三方WMS核心痛点
1. **定制化能力薄弱**：标准化产品无法适配多平台订单合并履约、批量备货、跨境退换货特殊校验、多库区分级管理等个性化场景，大量流程依赖人工线下补录，效率低、易出错。
2. **数据壁垒严重**：系统数据封闭，与订单中台、物流轨迹、财务结算系统对接不足，数据同步延迟、字段缺失，无法实现仓储、订单、物流、财务数据一体化联动，难以支撑精细化运营决策。
3. **运营成本偏高**：持续产生系统授权费、站点扩容费、接口定制费、运维服务费，业务扩张后成本逐年递增，可控性差。
4. **迭代响应滞后**：新增需求、流程优化需依赖厂商排期，周期长，无法匹配跨境业务快速迭代的节奏。
5. **权限与管控不足**：权限体系固定，无法适配公司多层级运营、审核、监管体系，库内操作追溯、风险管控能力薄弱。

### 2.3 自研WMS核心价值
1. **业务适配**：贴合跨境国内仓专属场景，支持流程配置与功能迭代，消除线下人工补录。
2. **数据打通**：深度对接各业务中台，实现订单、库存、履约、物流、财务数据实时互通，搭建全域仓储数据体系。
3. **成本优化**：替代付费第三方系统，长期降低采购与运维成本，自主掌控迭代节奏。
4. **管控升级**：搭建精细化库存、操作、权限、追溯体系，降低差错率，提升履约效率。
5. **可扩展性强**：支持多仓扩容、新场景叠加、自动化设备（PDA、分拣机）对接，适配未来业务增长。

## 3. 项目目标
### 3.1 业务目标
1. **风控底线：无感平稳切换**
依托四阶段灰度切流，完成全站点、全业务平滑迁移；切换期间业务不停服、订单无积压、履约无中断、数据无差异；异常场景支持熔断、回滚，履约时效与稳定性不低于原系统。

2. **落地目标：1:1平迁、等价复刻**
本次为纯系统替换，**不新增业务优化、不改造流程**；自研WMS 100%复刻原系统流程、功能、操作逻辑、校验规则，作业流程、操作习惯、承载能力保持一致，无业务退化。

3. **运营保障：作业无波动**
切换前后库内作业模式、人工成本、订单差错率持平，杜绝系统切换引发效率下降、错漏单、库存异常等问题。

4. **新仓目标：利用率持平**
完成新仓库位配置、库区规划、库存承载等全流程适配；库位使用率、吞吐能力、周转效率不低于原有仓库，无资源闲置、承载能力下降问题。

5. **基础治理：业务自主可控**
在平稳平移前提下，摆脱第三方系统依赖，实现仓储数据、流程、系统自主掌控，为后续迭代预留扩展空间；本次切换不做主动优化升级。

### 3.2 量化指标
- 履约完成率：48小时上网率 ≥ 90%；本次平迁不治理原有库存数据问题，仅保证**不新增、不放大库存差异**，杜绝履约阻断、订单异常。
- 新仓指标：库位使用率、吞吐效率、库区周转、业务承载规模与原仓库持平。

### 3.3 流量渗透目标
1. 自研订单渗透率
    - 入库：0% → 100%
    - 出库-备货：0% → 100%
    - 出库-拣选：比例待定，根据项目周期与风险评估后确定
2. 履约时效要求（不达标会导致平台封店）
    - 速卖通JIT：2点前付款订单，当日完成发货揽收，达标率100%
    - SHEIN订单：当日23点前完成发货揽收，达标率100%
    - 常规订单：前一日17点 ~ 当日17点推单订单，当日发货揽收；T+1发货达标率100%

## 5. 业务范围与边界
### 5.1 覆盖业务范围
#### 5.1.1 单量与时效说明
- **单量区间**：峰值13000单/日（120W pcs）；常规4000单/日（30W pcs）；近一月峰值10000单，最大发货量78W；每周周二为发货高峰。
- **履约时效**：严格遵循平台时效规则（速卖通JIT、SHEIN、常规订单时效要求见上文）。

#### 5.1.2 系统与业务划分
1. **系统**：第三方积水潭WMS、自研WMS；积水潭ERP、自研ERP
    - 优先切换已对接自研ERP的业务；小众平台（月单量＜1000）暂不切换，9月业务低谷期再规划。
2. **客户类型**：TOC、TOB
3. **电商平台**：LZD、AE、TK、SP、拼多多、1688、小众平台
4. **合作模式**：JIT、全托管
    - JIT、SHEIN全托管时效要求高、均包含标包加工，切换风险高，9月后再落地。
5. **订单规模**：大单、小单
6. **增值服务：标包加工**
    - 含标包加工：核心能力，自研WMS必须支持；
    - 无标包加工：TOC跨界小包已使用自研系统；混合增值服务订单暂走积水潭；
    - 1688 TOB小单（占积水潭订单70%）为本次主要切换对象，需解决自研WMS全链路标包适配问题。
7. **货型**：跨境小包、整箱货运

#### 5.1.3 全业务场景（国内仓全覆盖）
1. **头程/尾程**：OWM负责快递处理，异常线下处理
2. **仓库核心作业**
    - 基础资料：仓库、库区、库位、SKU、权限、操作规则配置

## 6. 核心业务流程切换需求
**核心原则**：纯平迁、等价替换、不优化、不升级，1:1复刻原系统流程、操作、校验规则；关键策略：逐节点梳理卡点与解决方案。

1. **切流顺序**：从发货链路开始切流，保证订单全链路完整，降低信息流复杂度。
2. **新仓方案**：利用新仓隔离双系统库存复杂度；采用「一盘货双系统」模式，通过缺货预警、仓间调拨完成系统调度，实物不移动。
3. **回滚兜底**：完整保留积水潭账号、产能、服务能力，支持一键回切；小众平台持续保留积水潭链路。
4. **风险优先**：优先完善人工作业能力，规避自动化作业单点故障。
5. **切流顺序**：先切标准能力 → 再切标包加工能力 → 最后优化人效。
6. **异常预案**：自研系统内置应急方案，支持纸质单打单、一键质检/复核，异常自动转人工，数据全程留痕。
7. **爬坡方案**
    - 方案一（本次目标）：一盘货模式，按库区圈仓配货，跨楼层自主调度，不改动现场，项目风险低；
    - 方案二（单独立项）：单层自闭环模式，适配小单、少SKU场景，仓利用率偏低，本次不纳入。

### 6.0 发货流程
当前商城订单**双发链路**：同时推送至积水潭、OMS；
切流依据订单透标（是否含标包加工）区分链路；异常场景支持快速回切至积水潭。

### 6.1 收货上架流程
1:1复刻第三方WMS全量逻辑、操作步骤、校验规则，本次不新增任何优化功能。

### 6.2 订单履约流程
完整复刻原有履约流程；保留积水潭兜底能力，新增人工防灾流程；
支持双向切换（积水潭 ↔ 自研WMS）、逐步灰度放量；
系统异常时自动切换人工模式，启用纸质单打单应急。

> 补充：积水潭按日出账、出库消耗结算，商务结算周期3天，决策前置5-7天；日均订单约9000单。

### 6.3 库存管理流程
仅优化影响发货时效、运营管理的内容，其余功能等价平移；保证发货率、仓利用率不低于原系统。
双系统库存互通，实物不动、系统调拨；支持缺货预警、仓内调拨预案。

### 6.4 盘点与调拨流程
完全复刻原有静态盘点、周期盘点、库存调整、跨区调拨规则；不新增差异比对、审核等优化功能。

### 6.5 退换货流程
1:1对齐退货核验、质检、入库、订单关联逻辑，不做流程简化、溯源优化，保证处理标准与原系统一致。

### 6.6 容量预案
系统承载不足时启用柔性承接能力，**优先保障JIT、SHEIN强时效订单**；其余订单控制发货时效。
入库、备货具备柔性能力，可通过工厂补货、批次HOLD单管控风险。

## 7. 上线切换与灰度切流方案
### 7.1 整体切换策略
采用**分阶段灰度、小步快跑、逐步放量**模式，拒绝一次性全量切换；
全程**双系统并行、流量可控、异常可回滚**，旧系统全程保留兜底能力。

### 7.2 四阶段灰度切流方案（纯平迁，不做优化）
核心原则：等价平移、稳字当头，仅验证自研WMS与原系统能力对齐，不测试新功能、不提效。

#### 7.2.1 第一阶段：标准基础链路灰度（基线跑通）
- 目标：规避自动化单点故障，验证人工模式下基础流程、链路、PDA稳定性；
- 切流范围：自动化仓**非标包加工**订单；
- 要求：双系统逐单数据核对，流程完整、数据一致；稳定运行3个工作日无重大问题，进入下一阶段。

#### 7.2.2 第二阶段：存量全功能对齐（能力验证）
- 目标：完善标包加工、异常预案能力，保证功能无缺失、逻辑无偏差；
- 切流范围：含标包加工类订单；
- 要求：全功能对标原系统，稳定运行5个工作日无退化，进入放量阶段。

#### 7.2.3 第三阶段：入库&备货全量切换
- 目标：完成全站点入库、备货链路切换；
- 切流范围：全站点、全楼层、全业务100%流量；
- 要求：稳定运行指定周期，账实相符、无履约积压，关停旧系统读写权限。

#### 7.2.4 第四阶段：单量大规模放量爬坡（单层放量）
- 目标：验证高单量场景下系统稳定性、性能，等价替代原系统；
- 切流范围：试点楼层/库区全量订单（常规单、备货、退换货、调拨等）；
- 要求：监控性能、接口、作业效率，解决高并发卡顿、数据同步问题；单层稳定运行7天，具备全楼层切换条件。

#### 7.2.5 第五阶段：全楼层全业务全覆盖（最终切换）
- 目标：彻底完成全仓库切换，替代第三方WMS；
- 切流范围：国内仓全楼层、全库区、全业务100%流量；
- 要求：长期稳定运行，账实一致、无故障，正式关停旧系统读写权限，项目切换闭环。

### 7.3 灰度通用管控规则
1. **流量可控**：支持比例调减、暂停、按库区/SKU/订单类型精准切流，随时回滚；
2. **异常熔断**：系统报错、数据异常、履约卡顿自动切回旧系统兜底；
3. **迭代兼容**：后续新功能上线同样遵循灰度流程。

### 7.4 切流节奏补充
各阶段时长、放量比例、峰值阈值、特殊规则，后续输出《灰度切流详细排期与放量单量方案》。

### 7.5 切换保障措施
1. 7*12小时现场+线上值守，快速处置突发问题；
2. 全员操作培训，确保仓管人员熟练使用自研系统；
3. 建立问题分级响应、限时闭环机制；
4. 双系统并行期间，每日核对库存、订单、单据数据。

### 7.6 验收标准
全部满足以下条件，判定项目验收完成：
1. 自研WMS100%复刻原系统功能，无缺失；
2. 全量数据迁移完成，单据可追溯；
3. 全业务流程顺畅，无卡顿、无积压、无新增异常；
4. 系统稳定运行15天无重大故障，量化指标全部达标；
5. 全员完成培训，可独立操作系统；
6. 操作手册、运维手册等全套文档归档完成。

### 7.7 风险与应对方案
| 风险场景 | 风险等级 | 应对方案 |
| ---- | ---- | ---- |
| 小流量切换后流程卡顿、履约中断 | 中 | 双系统并行、分步灰度，不直接关停旧系统；值守团队即时排障 |
| 大流量切换后流程卡顿、履约中断 | 高 | 1. 启用纸质单打单应急发货；2. 核验积水潭剩余额度，保留兜底能力 |
| 自研自动化仓操作异常 | 中 | 单点熔断，优先验证人工仓链路 |
| 新区域人效偏低 | 中 | 1. 优先处理强时效订单；2. 启用外挂存储区、仓间调拨补货 |

## 8. 项目整体计划与里程碑
项目整体周期：暂定 6.8 - 7.15（避开订单峰值）
整体阶段：需求 → 开发 → 测试 → 灰度试运行 → 全量上线 → 收尾复盘

### 8.1 项目进度规划
1. **交付优化**
    - 串行轮动替换，缩短周期；
    - 搭建测试仓，提前模拟单据、验证PDA、流程卡点；
    - 搭建备货仓，分段周转降低实施风险；
    - 研发分三期交付：一期（卡点功能）、二期（增值服务）、三期（人效优化）。
2. **进度风险**
    - 一、二期可并行研发+分库区实施，问题早暴露，风险可控；
    - 三期人效优化依赖前两期成果，高并发稳定性、应急预案要求高，7月落地难度大，时间待定。
3. **范围与时间锁定**
    - 一、二期范围&时间：预计6月15日锁定；
    - 三期范围&时间：预计6月26日锁定；
    - 切流时间结合爆单窗口、项目进度综合判定。
