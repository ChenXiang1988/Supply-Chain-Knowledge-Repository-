# xyWMS 售后入库 HTML 原型生成提示词

## 0. 文档信息

- 标题：xyWMS 售后入库 HTML 原型生成提示词
- 文档类型：原型提示词
- 版本：V1.0
- 日期：2026-06-19
- 作者：Martin
- 相关方：仓库运营、收货岗位、质检岗位、上架岗位、仓内主管、产品、研发、测试
- 来源文件：`20260618_Prototype_AfterSalesReceipt.md`

## 1. 任务目标

把 `20260618_Prototype_AfterSalesReceipt.md` 里的文字原型说明，转成一个可直接预览的 PC 端 WMS 后台 HTML 原型。

这不是文档展示页，也不是把 Markdown 原文排版成网页，而是一个真实的后台管理工作台原型。

## 2. 视觉与结构要求

- 页面类型：PC 端后台管理系统
- 视觉参考：接近标准 WMS / ERP 工作台风格，参考截图中的布局
- 整体结构：
  - 左侧深色导航栏
  - 顶部浅色系统栏
  - 内容区顶部页签
  - 状态 Tab
  - 查询筛选区
  - 操作按钮区
  - 数据表格区
  - 分页区
- 主视觉要求：
  - 不做成长文档页
  - 不把页面拆成大段说明卡片
  - 用业务系统表格、表单、状态标签、抽屉、弹窗表达交互
  - 风格克制、偏企业软件，不做营销站风格

## 3. 页面范围

HTML 原型至少包含以下 5 个工作台，并支持在同一页面内切换：

1. 到货签收单工作台
2. 质检入库单工作台
3. 上架单工作台
4. 异常待处理工作台
5. 仓库级售后入库配置

默认首屏展示：`到货签收单工作台`

  - 时间范围

  - 批量补关联
  - 查看异常
- 表格字段：

  - 到货形态
  - 创建方式
  - 来源识别状态
  - 售后关联状态

  - 操作人

  - 操作

## 5. 交互要求

- 原型需要包含基本前端交互：
  - 左侧菜单切换工作台
  - 顶部页签切换
  - 状态 Tab 切换高亮
  - 查询区展开 / 收起
  - 表格行 hover
  - 点击“详情”打开右侧抽屉
  - 点击“新建 / 补关联 / 标记异常 / 分段拆分 / 关闭异常 / 新增配置”等按钮打开弹窗
- 不需要真实后端
- 使用静态 mock 数据
- 交互重点是展示页面结构和业务动作，不是实现完整业务逻辑

## 6. 数据与文案约束

- 所有字段名称、状态名称、按钮名称要优先复用 Markdown 原型中的业务口径
- 到货签收单不能出现“转正常到货”相关动作
- 必须体现：
  - 待售后匹配
  - 已识别为售后
  - 待补关联
  - 无法识别
  - 阻断质检 / 放行质检
  - 质检确认数量
  - 上架只改库位不改数量

## 7. HTML 输出要求

- 输出形式：单个可直接打开的 `html` 文件
- 技术要求：
  - 纯 HTML + CSS + 原生 JS
  - 不依赖外部构建工具
  - 不依赖网络资源
- 代码要求：
  - 结构清晰
  - 样式集中
  - mock 数据集中
  - 交互脚本独立整理

## 8. 成功标准

以下条件全部满足才算生成成功：

1. 首屏看起来是 WMS 后台工作台，不是文档页
2. 布局和截图方向一致：左导航 + 顶栏 + 筛选 + 表格
3. 业务字段来自 Markdown 原型，而不是随意生成
4. 5 个工作台都能切换查看
5. 关键按钮能打开抽屉或弹窗，体现业务动作
