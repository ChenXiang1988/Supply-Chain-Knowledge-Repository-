# WMS = Warehouse Management System

`03Warehouse Management System` 用于沉淀 xzy 的仓储管理系统资料、方案和待办。

xzy 是我工作的其中一家企业，这是一家集工厂、仓储服务、ToB 销售、ToC 销售为一体的企业。由于仓储侧系统架构比较复杂，这里单独作为一个案例来记录。

## 面向用户
- 单仓管理者
- 单仓执行者
- 运维/测试支持

## 主要功能
- 单仓管理者：单据查询、作业节点监控、作业数据查询、任务进展查询、效能查询、异常汇报查询
- 单仓执行者：作业任务查询、异常汇报、单据打印、快递面单打印、作业标签打印

### 阶段 0 测试仓方案
- 目标：在测试环境和隔离测试区域内验证自研云仓 WMS 的核心功能
- 方法：Mock 数据驱动流程，纸盒 + SKU 标签模拟实物
- 核心量级：2 个货主、40 个 SKU、调拨入库不少于 6 批次、人工出库不少于 50 笔、全流程盘点 2 轮
- 边界：不接入真实订单、真实库存和正式切流
