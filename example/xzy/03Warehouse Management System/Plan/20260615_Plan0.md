## 一、方案背景
目前皓物仓业务依托聚水潭系统开展，为完成仓储体系迭代升级，后续需整体切换至自研云仓WMS体系。为保障正式切流工作平稳落地，规避线上业务风险，特启动**阶段0测试仓验证工作**。

本阶段通过搭建小型专属测试仓储场景，依托标准化Mock测试数据+轻量化实物模拟的方式，完整校验自研云仓WMS核心功能的可用性、流程适配性，排查系统Bug、流程卡点及仓储适配问题，为阶段1正式初步切流筑牢基础。

## 二、阶段0实施目标
依托皓物仓现有场地，划定**100-500㎡专属测试区域**，搭建标准化测试仓，基于Mock模拟数据+纸盒标签模拟实物的轻量化模式，完成自研云仓WMS入库、出库、盘点全流程业务闭环测试，验证系统基础功能与仓储实操流程的匹配度。

阶段核心测试规范与量级标准：
1. **测试SKU**：搭建多货主SKU体系，采用标准化Mock数据生成，贴合真实业务品类参数
2. **测试订单**：全部采用上游系统Mock推送订单，仅测试系统自动化非手工订单，校验全链路自动化流转能力
3. **作业模式**：**Mock数据驱动系统流程，粘贴SKU标签的纸盒模拟实物**，低成本还原真实仓储作业全链路
4. **测试量级**：全覆盖多业务场景，调拨入库覆盖3类核心场景、不少于6批次；人工出库不少于50笔、覆盖4类以上SKU组合；全流程盘点不少于2轮；所有场景均通过标准化Mock数据驱动，保证测试结果可落地、可参考，支撑后续正式切流

## 三、实施范围与核心边界
### 3.1 物理范围
皓物仓8F区域内划定100-500㎡独立区域，单独划分拣货区、备货区，与正式作业区域物理隔离，全程不影响仓库日常正式业务运转。

### 3.2 业务范围
仅开展自研云仓WMS系统功能测试，包含场地搭建、库位管理、SKU Mock数据初始化、Mock调拨入库测试、Mock人工出库测试、全流程盘点测试六大核心环节。**本阶段不涉及线上真实订单切换、不影响现有聚水潭正式业务流转**。

### 3.3 数据范围
所有测试单据、货主档案、SKU档案、订单信息、调拨信息均为**标准化Mock测试数据**，仅用于WMS功能验证，不接入线上真实业务数据、不占用正式业务数据资源。

### 3.4 实物边界
测试全程无需真实商品，采用「统一规格纸盒+专属SKU条码标签」模拟实物货品，仅验证作业流程与系统逻辑，不涉及真实商品仓储、配送环节。

### 3.5 风险边界
本阶段为纯测试场景，无正式业务流量接入，无需制定线上应急切换、故障兜底方案，仅针对测试场景数据异常、系统Bug、作业流程卡点做优化整改。

## 四、详细实施步骤
### 4.1 场地准备（前期筹备阶段）
联动皓物仓场地管理负责人，排查8F可用闲置区域，结合测试需求筛选100-500㎡平整、独立、动线合理的场地。完成场地确认、使用报备与审批，划定测试仓专属边界，明确区域权责，避免与正式作业混用，输出《测试场地确认表》。

### 4.2 场地标准化布置
基于选定场地完成功能分区搭建与标准化管理配置，打造合规测试作业场景：
1. **区域搭建**：分别部署拣货区货架、备货区货架，根据场地尺寸合理规划货架摆放间距、作业动线，保障实操便捷性。
2. **库位体系搭建**：制定统一库位编码规则，完成全区域库位划分、编码生成、标签打印与粘贴，确保每个库位编码唯一、清晰可辨。
3. **作业规则配置**：结合仓储作业规范，设定测试仓标准化拣货顺序、库位使用规则，同步录入自研云仓WMS系统，实现系统与现场作业规则统一。

### 4.3 SKU & 货主Mock数据初始化
本环节所有基础档案**全部采用标准化Mock数据构建**，统一字段规范、数据格式、参数标准，实现测试场景标准化、可复用、可追溯，贴合真实业务模型。
1. **基础数据配置**：搭建**2个独立货主Mock档案**，每个货主配置**20个标准SKU Mock数据**，合计40个测试SKU。SKU数据完整覆盖品类、规格、条码、重量、批次、保质期、基础库存等核心字段。
2. **数据关联绑定**：在自研云仓WMS系统录入Mock货主、Mock SKU完整资料，完成货主与SKU绑定、SKU与测试库位关联匹配。
3. **测试实物模拟**：无需真实商品，采用**统一规格纸盒+专属SKU条码标签**模拟实物货品。为每一个Mock SKU对应分配独立测试纸盒，纸盒表面粘贴对应SKU的条码标签，标签包含SKU编码、名称、条码等核心信息，保证系统Mock数据、测试纸盒、库位信息三者一一对应，可扫码识别，完全模拟真实货品作业流程。
4. **数据核验**：双人交叉校验所有Mock数据，确保无重复、无遗漏、无错配，数据准确率100%。

### 4.4 Mock调拨入库全流程测试
基于预制的标准化Mock调拨单据，结合纸盒模拟实物完成入库全流程测试，覆盖**全量到货、少到、多到**三类核心业务场景，验证系统入库模块功能稳定性与数据准确性。
1. **测试场景与单量标准**
    - 累计完成不少于6批次Mock调拨入库测试，100%覆盖三类核心场景：
      - 场景1：全量到货（实收数量=调拨单计划数量），共3批次，为基础常规业务场景
      - 场景2：少到（实收数量＜调拨单计划数量），共2批次，模拟调出仓短装差异场景
      - 场景3：多到（实收数量＞调拨单计划数量），共1批次，模拟调出仓溢装差异场景
    - 单批次调拨单覆盖10-20个SKU，单SKU调拨基数为10-20件，总模拟入库件数不低于300件；差异场景按±10%-30%比例设置实收差异量，贴合真实业务差异幅度。
2. **Mock数据生成要求**
    - 每张Mock调拨单包含唯一单据号、货主信息、来源仓库、SKU明细、计划调拨数量、预计到货时间、批次号等完整业务字段
    - 差异场景单据需提前预设计划数与实收数差值，同步匹配对应数量的测试纸盒，保证系统数据与实操数量完全对齐
    - 所有Mock调拨单的字段格式、数据结构、命名规则与线上正式业务单据1:1对齐，确保测试结果具备业务参考性
3. **实操流程**：通过Mock数据生成调拨入库任务，作业人员根据系统指引完成模拟货品收货、扫码核验、纸盒上架、库位绑定、差异登记、系统库存确认全流程操作。
4. **测试验证点**：校验Mock单据解析、任务下发、扫码入库、库存同步、差异自动记录、库位绑定功能，重点验证异常差异场景下系统的容错能力与处理逻辑，记录系统问题及流程卡点。

### 4.5 Mock人工出库测试
所有出库订单**完全模拟上游业务系统Mock自动推送**，零人工手动建单，纯自动化链路测试WMS出库全流程能力。
1. **测试单量与SKU组合标准**
    - 累计完成不少于**50笔**标准化Mock出库订单，覆盖至少4类SKU组合场景：
      - 组合1：单SKU单件订单（基础散单场景）
      - 组合2：单SKU多件订单（同品多件出库场景）
      - 组合3：2-3个SKU混合订单（常规多品出库场景）
      - 组合4：4个及以上SKU混合订单（多品凑单出库场景）
    - 单量分布：单SKU订单占比40%，多SKU组合订单占比60%；单笔订单商品总件数1-8件，全覆盖日常常规出库量级。
    - SKU覆盖要求：50笔订单需覆盖全部40个测试SKU的80%以上，避免单一SKU重复测试，保证场景全面性。
2. **Mock数据生成要求**
    - 每张Mock订单包含唯一订单号、货主信息、收件人基础信息、商品SKU明细、下单时间、配送类型、订单备注等完整业务字段
    - 订单字段格式、数据结构、接口协议与线上正式推送订单1:1对齐，确保测试兼容性与业务真实性
    - 50笔订单按预设SKU组合比例随机生成，同步设置不同的订单时效、备注要求等附加字段，模拟真实业务多样性
3. **实操流程**：上游批量推送Mock订单后，WMS自动解析、自动生成拣货任务、自动分配库位、优化拣货路径，作业人员按系统指引完成纸盒拣货、复核、打包、出库确认，系统自动扣减库存、留存出库全链路操作记录。
4. **测试验证点**：校验订单自动接入、任务自动分发、拣货路径优化、库存自动扣减、出库数据归档全链路自动化能力，重点验证多SKU组合订单的拣货、复核流程顺畅性与数据准确性。

### 4.6 全流程盘点测试（含Mock差异场景）
结合系统库存与纸盒模拟实物库存，完成全量盘点闭环测试，支持预制Mock差异数据，验证系统盘点差异处理能力。
1. **测试频次标准**：完成**2轮全量盘点**（1轮常规初盘+1轮差异复盘）。
2. **测试场景覆盖**：包含账实相符常规盘点、预制Mock账实差异盘点两种场景。
3. **实操流程**：系统生成盘点任务，人工对所有测试纸盒全盘清点，比对系统Mock库存数据，针对差异SKU完成复盘、差异登记、库存调整、数据闭环。
4. **测试验证点**：校验盘点任务生成、数据比对、差异记录、库存修正、复盘确认全流程功能。

## 六、风险识别与应对措施
| 风险场景 | 影响说明 | 应对措施 |
| --- | --- | --- |
| 场地协调延迟、区域规划不合理 | 延误整体测试进度，作业动线影响测试效果 | 提前对接仓管部门，优先锁定闲置区域，提前完成场地勘测与动线规划 |
| Mock货主/SKU数据错误、字段不规范 | 导致入库、出库测试数据失真，测试结果无效 | 统一Mock数据模板，录入完成后双人交叉核验，建立数据台账 |
| 系统无法正常解析上游Mock订单 | 人工出库链路中断，无法验证核心能力 | 提前对接研发适配Mock订单字段格式，预跑测试单据校验兼容性 |
| 调拨差异场景系统处理逻辑异常 | 无法验证异常入库能力，测试覆盖不全 | 提前预设差异标准值，测试全程跟进日志，发现问题即时修复复测 |
| 盘点Mock差异场景无法闭环 | 无法验证系统差异处理能力，测试覆盖不全 | 提前预制标准差异数据，复盘全程跟进，确保差异流程可闭环 |
| 系统功能卡顿、流程报错 | 中断测试流程，无法完成全流程验证 | 测试全程记录问题，实时同步研发团队，即时修复后复测，留存问题台账 |
