## 文档定位

### 1. 全球 3PL 平台蓝图

总纲文件，定义平台目标、设计原则、总体架构、区域适配和边界约束。

### 2. 业务能力树

把平台需要具备的能力拆成层级，明确哪些是基础底座，哪些是业务层能力，哪些是区域差异能力。

### 3. 核心领域模型

定义平台核心对象、关系、聚合边界、状态机和领域事件，统一数据语义。

### 4. 角色权限矩阵

定义内部团队、客户、合作方的权限边界、数据范围和高风险动作控制。

### 5. 主流程泳道图

描述订单到履约到结算的主链路，以及入库、出库、退货、保税等关键变体流程。

### 6. 国家 / 仓型配置清单

定义新国家和新仓型上线时必须配置的内容，支持海外仓快速复制。

### 7. 蓝图实现路径

把蓝图压缩成可执行的实施路线，明确先做什么、先验证什么、先复制什么。

### 8. 行业参考案例与模式

汇总可直接借鉴的行业样本，帮助实施路径和试点范围对齐市场主流做法。

## 使用方式

- 如果你在做战略讨论，先看总纲，再看能力树。
- 如果你在做产品拆解，先看领域模型，再看角色权限和主流程。
- 如果你在做新国家复制，先看国家 / 仓型配置清单。
