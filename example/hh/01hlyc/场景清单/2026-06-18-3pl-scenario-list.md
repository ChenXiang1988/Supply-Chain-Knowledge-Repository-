## 1. 目标

这份文档把 3PL 平台最关键的业务场景先列清楚，避免后续方案和页面只围着抽象概念转。

## 2. 场景总表

## 3. 第一阶段建议只做的场景

如果要先做一个最小可落地版本，建议只保留以下闭环：

1. 客户接入与账号开通
2. 主数据初始化
3. 入库预约与到货登记
4. 收货、验货、上架
5. 库存查询与占用
6. 出库下单与承诺
7. 拣货、复核、打包、发运
8. 异常工单与处置
9. 保税 / 清关单证处理

## 4. 不建议首批做的场景

| 场景 | 原因 |
|---|---|
| 多国家并行复制 | 会让规则复杂度快速上升 |
| 多仓型同时上线 | 会让配置验证难以收敛 |
| 全量报表中心 | 口径未稳定前，报表容易失真 |
| 过多的个性化字段 | 容易把第一版做成“定制泥潭” |
| 复杂的自动优化算法 | 先把规则跑通比优化更重要 |

## 5. 场景编写规则

后续每个场景都应该继续补齐这几个字段：

- 场景编号
- 场景名称
- 用户角色
- 触发条件
- 前置条件
- 主流程
- 异常 / 边界情况
- 场景结果
