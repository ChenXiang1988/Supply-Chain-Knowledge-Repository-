# 3PL 平台用户画像清单

文档类型：用户画像  
版本：V0.1  
日期：2026-06-18  
作者：Codex  
相关方：产品、架构、研发、仓储运营、关务、财务、客户成功

## 1. 目标

这份文档用来回答一个最现实的问题：这套 3PL 平台到底给谁用。

如果用户画像不清楚，后续的场景、页面、权限、数据口径都会散掉。

## 2. 用户分层

平台用户可以分成四层：

1. 内部总部与区域管理
2. 仓库现场执行
3. 客户 / 货主
4. 海外合作方 / 外部协同方

## 3. 画像清单

| 角色 | 主要目标 | 高频动作 | 最关注指标 | 使用终端 | 首批重要性 |
|---|---|---|---|---|---|
| 总部运营负责人 | 看全局仓网和业务健康度 | 看板、审批、规则调整、异常追踪 | SLA、库存准确率、履约时效 | Web | 高 |
| 区域负责人 | 管理国家/地区差异和伙伴协作 | 查看区域订单、库存、合作方绩效 | 区域履约率、成本、异常率 | Web | 高 |
| 仓库经理 | 保证现场作业稳定 | 分配任务、查看异常、管理库位和人员 | 出入库效率、现场准确率、异常关闭率 | Web + Handheld | 高 |
| 仓库作业员 | 按系统指令完成操作 | 收货、上架、拣货、复核、打包、发运 | 任务完成率、错误率 | Handheld | 高 |
| 合规 / 关务 | 保证单证和监管动作合规 | 看单证、审状态、处理申报和放行 | 放行时效、合规率、退单率 | Web | 高 |
| 财务 | 保证收费和对账准确 | 计费、对账、开票、核销 | 账单准确率、回款周期 | Web | 中高 |
| 客户 / 货主 | 看库存、下单、追踪、下载单证 | 下单、查库存、看轨迹、看异常、对账 | 可视化程度、响应时效、差错率 | Portal / API | 高 |
| 海外合作方 | 接单并回传状态 | 收货确认、作业回传、异常处理、对账 | 回传及时率、协同效率 | Portal / API | 中高 |

## 4. 首批优先用户

如果只选一批用户做试点，建议优先考虑：

1. 仓库经理
2. 仓库作业员
3. 客户 / 货主
4. 总部运营负责人

原因很简单：

- 他们覆盖最核心的执行和可视链路
- 他们能最快验证系统是否真的可用
- 他们最容易暴露流程、权限和异常问题

## 5. 画像使用方式

这些画像后续会直接影响：

- 场景清单
- 权限矩阵
- 页面原型
- 用例设计
- 培训材料

## 6. 待确认项

| 待确认项 | 为什么重要 |
|---|---|
| 首批试点是国内仓、海外仓还是保税仓 | 决定场景优先级和合规复杂度 |
| 首批客户是 B2B、B2C 还是混合 | 决定页面和履约方式 |
| 首批是否需要客户门户 | 决定是否把协同层纳入第一版 |
| 首批是否需要合作方回传 | 决定集成和权限设计 |
