| 优先级 | P0 |

## 2. 场景描述

上架完成后库存入账，客户需要在门户看到自己的库存状态——有多少可用、有多少被订单占用、有多少在途。仓库经理需要从仓库视角看库存分布（哪个库位有什么 SKU）。库存查询是客户门户的核心价值，也是出库下单的前置条件。

## 3. 场景主干

```
库存入账(S-04) → 库存流水记录 → 库存快照更新 → 客户门户查询 → 仓库侧库位库存查询
```

## 4. 子场景展开

### 4.1 子场景 A：客户查库存（门户）

| 字段 | 内容 |
|---|---|
| 角色 | 客户管理员 / 客户操作员 |
| 触发条件 | 客户需要了解库存状态 |
| 终端 | 门户 |

**页面结构：**

**4.1.1 库存概览卡片**

| 指标 | 数据来源 | 说明 |
|---|---|---|
| 可用库存 | 库存快照 - 已占用 - 冻结 | 可以下单的数量 |
| 已占用 | 订单承诺后未拣货的数量 | S-06 订单承诺时扣减可用量 |
| 在途入库 | 已预约但未上架的数量 | S-03 已确认→S-04 完成前的阶段 |
| 冻结库存 | 破损/质检待定数量 | S-04 差异处理标记为隔离 |

**4.1.2 SKU 库存明细列表**

| 列 | 说明 | 本期展示 |
|---|---|---|
| SKU 编码 | 客户侧商品编号 | 是 |
| SKU 名称 | 商品名称 | 是 |
| 条码 | 69码或厂家条码 | 是 |
| 可用库存 | 当前可下单数量 | 是 |
| 已占用 | 已被订单承诺的数量 | 是 |
| 在途入库 | 预约中的到货数量 | 是 |
| 库存单位 | 个/件 | 是 |

**4.1.3 筛选与排序（本期）**

| 功能 | 说明 |
|---|---|
| 按 SKU 搜索 | 输入编码或名称模糊搜索 |
| 按仓库筛选 | 本期仅一个仓库，多仓后扩展 |
| 按可用量排序 | 升序/降序 |
| 只看有库存 | 过滤掉可用量为0的 SKU |

**边界情况：**
- 库存为 0 的 SKU 正常显示，可用量显示 0
- 客户只能看到自己所绑定仓库的库存（数据权限由 S-01 仓库绑定决定）
- 页面刷新间隔不做实时推送（第二期做 WebSocket 实时更新）

**B2B vs B2C 库存视图差异（本期不区分，统一视图）：**

| 维度 | 未来扩展 |
|---|---|
| B2B 关注整箱/整托可用量 | 第二期做多包装单位 |
| B2C 关注散件可用量+效期 | 第二期做批次/效期管理 |
| B2B 可能需要查看库龄 | 第二期做库龄报表 |

### 4.2 子场景 B：仓库经理看库存分布（Web）

| 字段 | 内容 |
|---|---|
| 角色 | 仓库经理 |
| 触发条件 | 需要查看库位级库存 |
| 终端 | Web |

**页面：库位库存总览**

| 列 | 说明 |
|---|---|
| 库位编号 | 仓库结构配置的编号 |
| SKU 编码 + 名称 | 该库位存放的 SKU |
| 库存数量 | 该库位该 SKU 的件数 |
| 库位状态 | 正常 / 已满 / 空 |
| 最后操作时间 | 最近一次上架或拣货时间 |

**筛选：**
- 按楼层筛选（楼栋仓：1F-9F；平层仓：仅1F）
- 按库区筛选（存储区 / 拣货区）
- 按 SKU 搜索（看某个 SKU 分布在哪些库位）
- 只看非空库位

**边界情况：**
- 一个库位可以存放多个 SKU（混放），列表按 SKU 分行
- 一个 SKU 可以分布在多个库位，客户视图聚合展示，仓库视图按库位拆开展示

### 4.3 子场景 C：上架员/拣货员扫库位查库存（Handheld）

| 字段 | 内容 |
|---|---|
| 角色 | 上架员 / 拣货员 |
| 触发条件 | 作业过程中需要快速确认库位上的 SKU |
| 终端 | Handheld |

**操作流程：**
1. 作业员在 Handheld 上打开"库存查询"
2. 扫描库位条码（或手动输入库位编号）
3. 系统展示该库位上所有 SKU 及数量
4. 作业员可继续扫描 SKU 条码，系统高亮匹配行

**本期简化：**
- Handheld 仅做"扫库位→看SKU清单"，支持扫 SKU 高亮
- 不开放 Handheld 端库存调整（调拨、盘点在第二期）

### 4.4 库存数据模型（本期）

```
库存流水（事实来源）
  ├── id
  ├── sku_id
  ├── warehouse_id
  ├── location_id（库位）
  ├── change_type（上架入库 / 订单占用 / 取消占用 / 其他）
  ├── change_quantity（正=增加，负=减少）
  ├── related_order_id（关联的订单/到货签收单ID）
  └── created_at

库存快照（查询加速）
  ├── sku_id + warehouse_id（聚合维度）
  ├── available_qty（可用库存）
  ├── reserved_qty（已占用）
  ├── in_transit_qty（在途）
  └── updated_at
```

**库存变更链路：**

| 动作 | 流水记录 | 快照影响 |
|---|---|---|
| S-04 上架确认 | change_type = "上架入库" | available_qty +N |
| S-06 订单承诺 | change_type = "订单占用" | available_qty -N, reserved_qty +N |
| S-07 发运确认 | change_type = "发运出库" | reserved_qty -N |
| 订单取消（承诺后） | change_type = "取消占用" | available_qty +N, reserved_qty -N |

## 5. 与前后场景的关联

### 前置依赖 S-04
- 上架确认后写入库存流水 + 更新快照
- 未上架的商品不在"可用库存"中

### 后置支撑 S-06
- 出库下单时需校验可用库存
- 订单承诺时扣减 available_qty
- 客户下单前通常先查看库存

## 6. 场景结束判定

| 判定条件 | 系统状态 |
|---|---|
| 客户可在门户看到完整的库存概览和明细 | 库存可视 |
| 仓库经理可在 Web 看到库位级库存分布 | 库存可管理 |
| 库存流水数据完整，快照与实际一致 | 库存准确 |
| 订单承诺时可正确扣减可用库存 | 库存可用 |
