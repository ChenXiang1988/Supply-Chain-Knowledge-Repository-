# 3PL 平台权限矩阵设计

文档类型：权限设计  
版本：V0.1  
日期：2026-06-20  
作者：Martin  
相关方：产品、架构、研发、安全

## 1. 目标

本文档定义 3PL 平台的 RBAC（基于角色的访问控制）权限矩阵，明确：
- 每个角色能访问哪些端
- 每个角色能操作哪些功能模块
- 每个角色能执行哪些操作（查看、创建、编辑、删除、审批）

避免后续开发中"权限混乱、数据泄露、操作失控"。

## 4. 数据权限规则

### 4.1 数据隔离原则

| 数据类型 | 隔离规则 | 实现方式 |
|---|---|---|
| **客户数据** | 客户只能看到自己的数据 | 按 `client_id` 过滤 |
| **仓库数据** | 仓库经理/员工只能看到本仓库数据 | 按 `warehouse_id` 过滤 |
| **订单数据** | 客户只能看到自己的订单 | 按 `client_id` 过滤 |
| **库存数据** | 客户只能看到自己的库存 | 按 `client_id` 过滤 |
| **财务数据** | 客户只能看到自己的账单 | 按 `client_id` 过滤 |

### 4.2 字段级权限

| 字段类型 | 可见角色 | 说明 |
|---|---|---|
| **成本字段**（仓储费、操作费） | 总部运营、财务、客户管理员（仅自己） | 仓库执行角色不可见 |
| **利润字段** | 总部运营、财务 | 客户成功不可见 |
| **系统配置字段** | 总部运营 | 其他角色不可见 |

## 5. 操作权限定义

| 权限类型 | 说明 | 适用场景 |
|---|---|---|
| **查看 (Read)** | 查看数据，不能修改 | 报表、监控看板 |
| **创建 (Create)** | 新建数据 | 创建订单、创建预约 |
| **编辑 (Update)** | 修改数据 | 编辑SKU信息、编辑订单 |
| **删除 (Delete)** | 删除数据（逻辑删除） | 删除订单（需审批） |
| **审批 (Approve)** | 审批他人操作 | 审批预约、审批RMA |
| **执行 (Execute)** | 执行现场操作 | 拣货、打包、上架 |
| **导出 (Export)** | 导出数据 | 导出报表、导出账单 |

## 6. 特殊权限规则

### 6.1 审批链设计

| 操作类型 | 申请人 | 审批人 | 说明 |
|---|---|---|---|
| 删除订单 | 客服、客户操作员 | 仓库经理、总部运营 | 需填写删除原因 |
| 调整库存（>1000元） | 库存管理员 | 仓库经理 | 需上传证明材料 |
| 豁免计费 | 客户成功 | 总部运营 | 需填写豁免原因 |
| 修改系统配置 | 仓库经理 | 总部运营 | 需影响评估 |

### 6.2 敏感操作审计

以下操作必须记录审计日志：
- 删除订单/库存调整/计费规则修改
- 查看客户成本数据
- 导出客户数据
- 修改系统配置

审计日志内容：`谁、在什么时间、对什么数据、做了什么操作、操作前的值、操作后的值`

## 7. 权限矩阵实施建议

### 7.1 第一阶段（MVP）
- [ ] 实现基于角色的菜单权限（哪些角色能看到哪些菜单）
- [ ] 实现基于角色的操作权限（哪些角色能执行哪些操作）
- [ ] 实现数据隔离（按 `client_id`、`warehouse_id` 过滤）

### 7.2 第二阶段（增强）
- [ ] 实现字段级权限（哪些角色能看到哪些字段）
- [ ] 实现审批链（需要审批的操作）
- [ ] 实现审计日志（敏感操作记录）

### 7.3 第三阶段（高级）
- [ ] 实现数据范围权限（如：仓库经理能看到本仓库+下级仓库）
- [ ] 实现时间维度权限（如：只能查看最近30天数据）
- [ ] 实现操作频率限制（如：同一操作1分钟内最多执行10次）
