## 1. 场景定位

| 项目 | 内容 |
|---|---|
| 场景编号 | S-01 |
| 场景名称 | 客户接入与账号开通 |
| 所属链路 | 基础配置链路 |
| 前置场景 | 无（平台起点） |
| 后置场景 | S-02 主数据初始化 |
| 优先级 | P0 |

## 2. 场景描述

新客户签约后，总部运营在系统中完成客户入驻：建立客户档案（组织）、按角色开通账号、分配仓库权限。这是客户使用平台的入口场景，没有这一步后面所有场景都无法运转。

## 3. 场景主干

```
客户签约 → 创建客户组织 → 配置初始角色 → 开通管理员账号 → 绑定仓库 → 开通门户
```

## 4. 子场景展开

### 4.1 子场景 A：创建客户组织

| 字段 | 内容 |
|---|---|
| 角色 | 总部运营 |
| 触发条件 | 新客户签约完成，合作范围已确认 |
| 终端 | Web |

**操作流程：**

1. 总部运营在"客户管理"中点击"新增客户"
2. 填写客户档案：

| 信息项 | 说明 | 必填 |
|---|---|---|
| 客户名称 | 公司全称 | 是 |
| 客户简称 | 系统内显示的短名称 | 是 |
| 客户类型 | B2B / B2C / 混合 | 是 |
| 统一社会信用代码 | 用于合同和账单 | 否 |
| 联系人姓名 | 客户方主要对接人 | 是 |
| 联系人手机 | 用于账号激活通知（本期线下通知） | 是 |
| 联系人邮箱 | 门户登录账号 | 是 |
| 结算币种 | 人民币（国内仓默认） | 否 |
| 备注 | 合作范围、特殊要求 | 否 |

3. 提交后系统创建客户组织，生成唯一的客户 ID
4. 客户状态 = "已启用"

**边界情况：**
- 客户名称重复时系统提示但不拦截（允许同名客户存在，以 ID 区分）
- 邮箱已被其他客户使用时拦截（一个邮箱只能绑定一个客户组织）

**本期处理：**
- 第一期不做 SSO / API Key 自动生成，客户门户用账号密码登录
- 激活通知走线下（运营人员邮件/微信发送临时密码）

### 4.2 子场景 B：开通管理员账号

| 字段 | 内容 |
|---|---|
| 角色 | 总部运营 |
| 触发条件 | 客户组织已创建 |
| 终端 | Web |

**操作流程：**

1. 系统在客户组织创建时自动生成一个初始管理员账号
2. 账号规则：

| 属性 | 规则 |
|---|---|
| 登录账号 | 联系人邮箱 |
| 初始密码 | 系统随机生成 8 位（含大小写+数字） |
| 绑定角色 | "客户管理员"（预置角色，每个客户组织至少一个） |
| 初始状态 | "待激活" |

3. 总部运营将初始密码线下发送给客户联系人
4. 客户首次登录门户时强制修改密码，账号状态变为"已激活"

**客户管理员权限范围（本期）：**

| 可操作 | 不可操作 |
|---|---|
| 修改本组织信息（非关键字段） | 修改客户类型 |
| 创建本组织下的子账号（仓库操作员等） | 绑定/解绑仓库 |
| 查看本组织订单、库存、轨迹 | 查看其他客户数据 |
| 提交入库预约、创建出库订单 | 修改费率、合同条款 |

**边界情况：**
- 客户管理员可自行新增子账号，但子账号角色范围受限于系统预置的客户侧角色
- 子账号只能看到本组织数据（数据权限在 S-01 完成后即生效）

### 4.3 子场景 C：绑定仓库

| 字段 | 内容 |
|---|---|
| 角色 | 总部运营 |
| 触发条件 | 客户组织已创建，合作范围中已确定使用哪些仓库 |
| 终端 | Web |

**操作流程：**

1. 总部运营在客户详情页中点击"仓库授权"
2. 从仓库列表中选择该客户可使用的仓库：

| 本期可选范围 |
|---|
| 国内普通仓（首批仅一个试点仓） |

3. 勾选仓库后确认
4. 系统记录客户-仓库绑定关系，后续该客户的所有预约、订单只能发往已绑定的仓库

**边界情况：**
- 客户解绑仓库前需确保该仓库内无未完成的订单和库存（本期不做强校验，第二期补）
- 同一客户可绑定多个仓库（本期仅一个试点仓）

### 4.4 子场景 D：开通门户访问

| 字段 | 内容 |
|---|---|
| 角色 | 总部运营（配置）/ 客户管理员（首次登录） |
| 触发条件 | 客户管理员账号已生成 |
| 终端 | 门户 |

**操作流程：**

1. 客户管理员收到初始密码后，访问门户登录页
2. 输入邮箱 + 初始密码
3. 系统检测到"待激活"状态，跳转到密码修改页
4. 客户设置新密码（规则：至少8位，含大小写+数字）
5. 密码修改成功，账号激活
6. 客户进入门户首页（本期首页展示：已绑定仓库列表、快捷入口）

**本期门户首页内容：**

| 模块 | 内容 |
|---|---|
| 我的仓库 | 已授权仓库列表（本期仅一个） |
| 快捷入口 | 入库预约、出库下单、查库存 |
| 最近动态 | 最近5条订单/预约状态变更（第二期做） |

## 5. 角色与权限边界

S-01 完成后，系统中应存在以下角色实例：

| 角色 | 归属 | 本期主要权限 |
|---|---|---|
| 总部运营管理员 | 平台 | 创建客户组织、绑定仓库、查看全量数据 |
| 客户管理员 | 客户组织 | 管理本组织子账号、提交预约/订单、查库存 |
| 仓库经理 | 平台（仓库维度） | 管理仓库配置、确认预约、审批差异（S-02→S-07） |
| 班组长（入库） | 平台（仓库维度） | 分派收货/验货/上架单、审批差异 |
| 班组长（出库） | 平台（仓库维度） | 分派拣货/复核/打包/发运任务 |
| 收货员 | 平台（仓库维度） | 到货登记、收货确认 |
| 验货员 | 平台（仓库维度） | 逐箱验货、差异登记 |
| 上架员 | 平台（仓库维度） | 上架入库、库位确认 |
| 拣货员 | 平台（仓库维度） | 拣货执行 |
| 复核员 | 平台（仓库维度） | 逐件复核 |
| 打包员 | 平台（仓库维度） | 打包、标签打印 |
| 发运员 | 平台（仓库维度） | 发运确认 |
| 叉车工 | 平台（仓库维度） | 楼层转运、大件搬运 |
| 库存管理员 | 平台（仓库维度） | 盘点、库存调整 |

## 6. 场景结束判定

| 判定条件 | 系统状态 |
|---|---|
| 客户组织已创建且状态为"已启用" | 客户档案可用 |
| 客户管理员账号已激活（首次登录并修改密码） | 客户可自主登录门户 |
| 至少一个仓库已绑定 | 客户可发起入库预约 |
| 门户首页可正常访问 | 客户接入流程完整闭环 |

## 8. 本期试点边界

| 本期做 | 本期不做 |
|---|---|
| 总部运营手动创建客户组织 | 客户自助注册 |
| 邮箱+密码登录门户 | SSO / API Key 认证 |
| 仓库绑定（单仓） | 多仓同时绑定 |
| 初始管理员账号自动生成 | 客户管理员自行创建组织 |
| 线下发送初始密码 | 自动发送激活邮件/短信 |
| 客户管理员管理本组织子账号 | 子账号审批流程 |
