## 📌 文档定位

### 基础定义类
这些文档定义平台的基础概念、用户角色、业务边界等，是所有场景需求的基础。

- **用户画像**: 定义谁会使用这个系统，他们的职责是什么
- **场景清单**: 列出所有业务场景，帮助理解平台覆盖范围
- **系统约束**: 明确平台能做什么、不能做什么
- **权限矩阵**: 定义不同角色的权限边界
- **产品端划分**: 定义系统的产品形态（Web、PDA、API等）

### 场景需求类
这些文档详细描述每个业务场景的功能要求，包括：
- 场景描述
- 用户故事
- 业务流程
- 功能要求
- 数据要求
- 非功能性要求

- **产品经理**: 重点关注所有文档，特别是场景需求文档
- **架构师**: 重点关注"基础定义类"文档，了解业务边界
- **研发**: 重点关注场景需求文档，了解功能要求
- **测试**: 重点关注场景需求文档，作为测试依据
- **实施**: 参考场景需求文档，了解系统功能

## 📝 文档模板

场景需求文档建议包含以下章节：

1. **场景概述**: 简要描述本场景
2. **用户故事**: 用户视角的使用场景
3. **业务流程**: 详细的业务流程图（建议使用Mermaid）
4. **功能要求**: 详细的功能点列表
5. **数据要求**: 涉及的数据对象、字段、状态等
6. **非功能性要求**: 性能、安全、可用性等要求
7. **验收标准**: 明确的质量标准
