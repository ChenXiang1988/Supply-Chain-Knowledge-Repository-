| 项目 | 内容 |
|---|---|
| 场景编号 | S-02 |
| 场景名称 | 主数据初始化 |
| 所属链路 | 基础配置链路 |
| 前置场景 | S-01 客户接入与账号开通 |

| 优先级 | P0 |

## 2. 场景描述

客户接入（S-01）完成后，还需要把业务能跑起来的基础数据配齐：仓库经理完成仓库物理结构配置（楼层/库区/库位），客户在门户完成 SKU 注册和贴码准备，总部运营配置客户服务规则和库区类型模板。配完之后，客户才能发起入库预约、仓库才能接收货物。

## 3. 场景主干

```
仓库结构配置 → SKU 主数据导入 → 客户服务规则配置 → 校验完整性 → 上线就绪
```

## 4. 子场景展开

### 4.1 子场景 A：库区类型模板定义

| 字段 | 内容 |
|---|---|
| 角色 | 总部运营 |
| 触发条件 | 平台初始化（仅需配置一次） |
| 终端 | Web |

**操作流程：**

总部运营在平台上定义标准库区类型模板，规定每个仓库必须配置的库区类型。仓库经理不能自己发明库区类型，只能在模板范围内做具体规划。

| 库区类型 | 说明 | 是否必配 |
|---|---|---|
| 收货区 | 到货后临时存放区域 | 是 |
| 验货区 | 验货操作区域（可与收货区合并） | 否 |
| 存储区 | 上架后长期存放区域 | 是 |
| 拣货区 | 高频 SKU 的拣选区域 | 否 |
| 发货区 | 打包后待发运区域 | 是 |
| 不合格品区 | 破损/错货隔离区域 | 是 |

### 4.2 子场景 B：仓库结构与库位配置

| 字段 | 内容 |
|---|---|
| 角色 | 仓库经理 |
| 触发条件 | 新仓库上线，库区类型模板已由总部定义 |
| 终端 | Web |

#### 4.2.1 仓库层级模型

平台统一采用"仓库 → 楼层 → 库区 → 库位"四级结构。平层仓和楼栋仓用同一套模型：

| 场景 | 楼层配置 | 效果 |
|---|---|---|
| 平层仓 | 1 层，默认命名为"1F" | 仓库经理不需要额外配置楼层，系统自动创建 1 层 |
| 楼栋仓 | N 层，按实际编号（1F-9F） | 仓库经理先配置楼层，再在每个楼层下配置库区 |

**操作流程：**

1. 仓库经理进入"仓库管理"，选择目标仓库
2. 配置仓库基础信息：

| 配置项 | 说明 | 本期必填 |
|---|---|---|
| 仓库名称 | 运营中使用的名称 | 是 |
| 仓库地址 | 省/市/区/详细地址 | 是 |
| 仓库类型 | 平层仓 / 楼栋仓 | 是 |
| 楼栋层数 | 当选择"楼栋仓"时必填（如 9） | 条件必填 |
| 是否有货梯 | 当选择"楼栋仓"时必填 | 条件必填 |
| 仓型 | 国内普通仓（本期唯一选项） | 是 |
| 运营模式 | 自营（本期唯一选项） | 是 |
| 服务对象 | B2B+B2C 混合（本期选项） | 是 |
| 时区 | UTC+8 | 是 |
| 联系人 + 电话 | 仓库现场对接人 | 是 |

3. 配置楼层（楼栋仓）：

| 楼层编号 | 楼层别名 | 楼层功能描述 | 是否货梯可达 |
|---|---|---|---|
| 1F | 收货层 | 收货区 + 验货区 + 不合格品区 | 是 |
| 2F | 发货层 | 打包区 + 发货区 | 是 |
| 3F | 拣货层 | 高频 SKU 拣选区 | 是 |
| 4F-9F | 存储层 | 按流速度/品类分区存储 | 是 |

4. 在每个楼层下，按总部定义的库区类型模板配置具体库区：

| 楼层 | 库区类型 | 库区名称示例 |
|---|---|---|
| 1F | 收货区 | 1F-收货区-1号门 |
| 1F | 验货区 | 1F-验货区 |
| 1F | 不合格品区 | 1F-不合格品区 |
| 3F | 拣货区 | 3F-拣货区-A区 |
| 4F | 存储区 | 4F-存储区-快流品 |
| … | … | … |

5. 配置库位（在每个库区下）：

**库位编码规则（加入楼层前缀）：**

| 仓库类型 | 编码示例 | 含义 |
|---|---|---|
| 平层仓 | A-01-01 | A区-第1排-第1位 |
| 楼栋仓 | 4F-A-01-01 | 4楼-A区-第1排-第1位 |

**本期处理：**
- 第一期采用手动编号，仓库经理按现场实际编号录入系统
- 平层仓默认只有 1 层，库位编号不含楼层前缀
- 楼栋仓库位编号强制包含楼层前缀（系统自动拼接，仓库经理只需填"区-排-位"）
- 批量导入：支持 Excel 模板批量上传库位清单（楼栋仓模板含楼层列）
- 库位属性（尺寸、承重、品类限制）本期不强制，留作第二期

**边界情况：**
- 同一仓库内库位编号不能重复
- 楼栋仓的库区必须先选楼层，不能跨楼层配置库区
- 库区删除前需确保该库区内无库存（本期不做强校验）
- 已上线的仓库修改楼层或库区结构需仓库经理确认
- 已上线的仓库修改库区结构需仓库经理确认

### 4.3 子场景 C：SKU 主数据注册与贴码管理

| 字段 | 内容 |
|---|---|
| 角色 | 客户管理员 |
| 触发条件 | 客户已完成 S-01，且有待入库的商品 |
| 终端 | 门户 |

**设计原则：**

SKU 编码是客户维度下的唯一标识，收货员在 Handheld 上扫到条码或 SKU 码后，系统在当前预约的客户上下文中定位到唯一商品。条码可以跨客户重复（一品多商），不要求全局唯一。

**操作流程：**

1. 客户管理员登录门户，进入"商品管理"
2. 新增 SKU：

**方式一：手动逐条添加**

| 字段 | 说明 | 必填 |
|---|---|---|
| SKU 编码 | 客户侧商品编号，本客户内唯一 | 是 |
| SKU 名称 | 商品名称 | 是 |
| 条码 | 商品69码或厂家条码 | 否（无条码时可留空） |
| 品类 | 按客户行业分类（如服装/3C/日化） | 否 |
| 包装单位 | 个/件 | 是 |
| 单件尺寸（cm） | 长×宽×高 | 否（第二期做库位推荐用） |
| 单件重量（kg） | 毛重 | 否（第二期做库位推荐用） |
| 有效期声明 | 有固定保质期 / 无保质期 / 不适用 | **本期必填** |
| 保质期（月） | 当选择"有固定保质期"时必填，如 12、24 | 条件必填 |
| 是否需要序列号管理 | 是/否（高价值商品建议启用） | 否 |

**方式二：Excel 模板批量导入**

| 步骤 |
|---|
| 1. 下载系统提供的 Excel 模板（含上述字段列） |
| 2. 按模板填写 SKU 清单 |
| 3. 上传文件 |
| 4. 系统逐行校验（必填项、SKU 编码是否重复） |
| 5. 校验通过的行导入，失败的行标红提示修改后重新上传 |

3. 导入完成后，SKU 状态 = "已启用"

**4.3.1 贴码规则**

| 场景 | 处理方式 |
|---|---|
| SKU 有条码，且条码未被其他客户使用 | 客户无需额外贴码，收货员扫描商品自带条码即可识别 |
| SKU 有条码，但该条码已被其他客户使用（一品多商） | 系统为该 SKU 自动生成内部码（格式：`HH-{客户ID简写}-{序号}`），并提供标签模板供客户下载打印。**客户必须在入仓前在每个最小包装上贴好系统生成的标签。未经贴码的货物仓库有权拒收。** |
| SKU 无条码 | 同上，系统自动生成内部码，客户必须在入仓前贴好标签 |

**标签模板内容：**

| 元素 | 说明 |
|---|---|
| SKU 内部码（一维码/二维码） | 系统生成，收货员扫描用 |
| SKU 编码 | 可读文本 |
| SKU 名称 | 可读文本 |
| 客户简称 | 可读文本 |

**本期处理：**
- SKU 编码在客户维度下唯一
- 条码允许跨客户重复，系统不拦截
- 包装层级本期仅支持单层（个/件），箱/托等包装层级第二期做
- 一品多商和无条码场景下，客户必须在门户下载标签模板、打印并贴码后，才能在 S-03 提交入库预约

**边界情况：**
- SKU 编码在本客户内重复时导入失败
- 有效期声明为必填，客户不可跳过
- 空行自动跳过，不报错
- 标签模板提供 PDF 和图片两种下载格式

### 4.4 子场景 D：客户服务规则配置

| 字段 | 内容 |
|---|---|
| 角色 | 总部运营 |
| 触发条件 | 客户 SKU 主数据已导入 |
| 终端 | Web |

**操作流程：**

在各客户的详情页中，配置以下服务规则：

**4.3.1 入库规则**

| 配置项 | 选项 | 本期默认值 |
|---|---|---|
| 是否允许无预约到货 | 是/否 | **否**（必须预约） |
| 入库预约提前量 | N小时/N天 | **提前 4 小时** |
| 是否按 ASN 严格比对 | 是/否 | **是**（S-04 验货以 ASN 为基准） |
| 超量收货上限 | 百分比 | **110%**（最多允许多收10%） |
| 少收是否计差异 | 是/否 | **是** |

**4.4.2 库存规则**

| 配置项 | 选项 | 本期默认值 |
|---|---|---|
| 库存分配策略 | 先进先出（按到期日） | **先进先出（按到期日）** |

**4.4.3 出库规则**

| 配置项 | 选项 | 本期默认值 |
|---|---|---|
| 截单时间 | HH:MM | **16:00**（当日16点后订单次日处理） |
| 是否支持合并拣货 | 是/否 | **是** |
| 出库优先级 | 按订单时间 / 按客户备注 | **按订单时间** |

**4.3.4 客户凭证配置（本期）**

| 配置项 | 选项 |
|---|---|
| 收货标签模板 | 默认模板（含 SKU、数量、库位） |
| 出库装箱单模板 | 默认模板（含 SKU、数量、收件人） |

**边界情况：**
- 规则配置与客户绑定，不同客户可以有不同规则
- 规则修改后，只对修改后新生成的订单/预约生效，已执行中的不受影响
- 规则变更记录应留存日志（第二期补）

### 4.5 子场景 E：初始化校验

| 字段 | 内容 |
|---|---|
| 角色 | 总部运营（操作）/ 系统（自动校验） |
| 触发条件 | 客户首次提交入库预约前 |

**校验清单：**

| 校验项 | 未通过时的提示 |
|---|---|
| 客户组织已创建 | "请先完成客户接入（S-01）" |
| 至少一个仓库已绑定 | "暂无可用仓库，请联系运营" |
| 仓库至少有一个存储区和一个收货区 | "仓库结构不完整，请补充库区" |
| 楼栋仓每个楼层至少配置了一个库区 | "楼层结构不完整，请为每层配置库区" |
| 该客户下有至少一个 SKU | "请先导入商品主数据" |
| 客户服务规则已配置 | "请先完成服务规则配置" |

**校验通过后：**
- 系统标记该客户/仓库为"可运营"状态
- 客户门户的"入库预约"和"出库下单"入口从灰色变为可点击

## 5. 与前后场景的关联

### 前置依赖 S-01
- 客户组织必须已创建
- 仓库经理、班组长、收货员、验货员、上架员、拣货员、复核员、打包员、发运员、叉车工账号必须已开通（S-01-D 子账号管理）
- 客户管理员账号已激活

### 后置支撑 S-03
- S-03 客户提交入库预约时，预约页面应自动拉取该客户下的 SKU 列表
- S-03 选择的仓库必须是 S-02 已完成结构配置的仓库
- S-04 收货扫描时，系统按条码匹配 SKU（依赖 S-02 导入的条码）

## 8. 本期试点边界

| 本期做 | 本期不做 |
|---|---|
| 仓库类型选择：平层仓 / 楼栋仓 | 楼栋仓货梯调度算法 |
| 总部定义库区类型模板，仓库经理在模板内配置 | 仓库经理自定义库区类型 |
| 手动编码库位（楼栋仓含楼层前缀） | 库位属性（尺寸、承重、品类限制） |
| 客户门户自行注册 SKU + 模板导入 | API 同步 SKU |
| 一品多商 / 无条码场景下系统生成内部码 + 客户贴码 | 客户不贴码即可入仓 |
| 有效期声明（必填）+ 保质期注册 | 批次号 / 序列号管理 |
| 单层包装单位 | 多层包装层级（箱/托） |
| 默认服务规则配置 | 自定义规则模板 |
| 线下校验 + 系统完整性检查 | 自动初始化工单流程 |
| 基础字段 SKU 主数据 | 商品图片、多语言名称、HS Code |
| 先进先出（按到期日）拣货策略 | 人工指定批次
