# 3PL 平台产品端划分

文档类型：产品架构  
版本：V0.1  
日期：2026-06-20  
作者：Martin  
相关方：产品、架构、研发、UX设计

## 1. 目标

本文档明确 3PL 平台的产品端划分，定义每个端的：
- 目标用户与使用场景
- 核心功能清单
- 与其他端的数据关联关系

避免后续设计和开发中出现"端职责不清、功能重复、数据孤岛"。

## 2. 端划分总览

平台共划分为 **6 个端**：

| 端名称 | 用户角色 | 使用设备 | 核心目标 |
|---|---|---|---|
| **Web管理后台** | 运营管理员、客服、财务、仓库经理 | PC浏览器 | 全流程管理、配置、监控 |
| **PDA移动作业端** | 仓库操作员工（收货员、验货员、上架员、拣货员、复核员、打包员、发运员、叉车工） | 手持PDA/工业手机 | 现场作业执行 |
| **客户门户Web端** | 客户管理员、客户操作员 | PC浏览器 | 自助服务（下单、查询、账单） |
| **开放平台API** | 电商平台、ERP系统、物流商系统 | 系统对接 | 数据自动同步 |
| **数据看板/BI端** | 总部运营、区域负责人、客户 | PC/大屏 | 实时监控、数据分析 |
| **退货处理Web端** | 质检员、客服、财务人员 | PC浏览器 | 退货RMA处理、决策 |

## 5. 端到端流程图

### 5.1 Web管理后台 - 订单处理全流程

```
客户下单
  ↓
订单接收（手动/API自动导入）
  ↓
订单校验（地址、库存、合规）
  ↓
[异常?] ──是──→ 异常工单创建 → 客服处理 → 返回订单校验
  ↓ 否
波次分配（按物流商、目的地、截单时间）
  ↓
任务下发至PDA
  ↓
[监控] ← 实时跟踪作业进度
  ↓
拣货完成确认
  ↓
复核完成确认
  ↓
打包完成确认
  ↓
面单打印与物流交接
  ↓
Tracking Number回传平台
  ↓
订单完成
```

---

### 5.2 PDA移动作业端 - 拣货作业流程

```
仓库员工登录PDA
  ↓
权限验证
  ↓
首页查看待办任务
  ↓
领取拣货任务
  ↓
[任务类型?]
  ├─ 波次拣货 → 显示波次商品清单
  └─ 单订单拣货 → 显示单个订单商品
  ↓
开始路径导航
  ↓
到达第一个库位
  ↓
扫描库位条码
  ↓
[库位正确?] ──否──→ 返回路径导航
  ↓ 是
扫描商品SKU条码
  ↓
[商品正确?] ──否──→ 提示错误: 商品不匹配
  ↓ 是
录入拣货数量
  ↓
[数量正确?] ──否──→ 触发缺货/异常处理
  ↓ 是
确认拣货
  ↓
[还有下一个商品?]
  ├─ 是 ──→ 返回路径导航
  └─ 否 ──→ 拣货完成
  ↓
包裹复核区交接
  ↓
任务结束
```

### 5.3 客户门户Web端 - 自助下单流程

```
客户登录门户
  ↓
创建订单（手动录入/批量导入/平台同步）
  ↓
SKU校验（是否存在、是否合规）
  ↓
[SKU异常?] ──是──→ 提示错误，返回修改
  ↓ 否
地址校验（格式、有效性）
  ↓
[地址异常?] ──是──→ 提示错误，返回修改
  ↓ 否
运费预估（按重量、目的地、物流商）
  ↓
确认下单
  ↓
订单提交至Web管理后台
  ↓
返回订单号与预计发货时间
```

---

### 5.4 开放平台API - 订单同步流程

```
平台推送订单（Webhook/轮询）
  ↓
接收订单数据
  ↓
数据清洗与去重
  ↓
[订单格式正确?] ──否──→ 记录错误日志，返回错误
  ↓ 是
写入WMS数据库
  ↓
触发订单校验（地址、库存、合规）
  ↓
[校验通过?] ──否──→ 创建异常工单，通知客服
  ↓ 是
订单进入可分配状态
  ↓
返回接收成功状态至平台
```

### 5.5 退货处理Web端 - RMA全生命周期流程

```

  ↓ 是

  ↓

  ↓
客户寄回商品
  ↓
仓库收货（扫描RMA号）
  ↓
退货质检（质检员）
  ↓
[质检结果?]
  ├─ 可售 ──→ 重新上架 → 库存更新
  ├─ 需维修 ──→ 生成维修工单 → 维修后重新上架
  └─ 需销毁 ──→ 生成销毁记录 → 合规处理
  ↓
通知客户退货处理结果
  ↓
RMA关闭
```

## 6. 端优先级建议（MVP阶段）

| 端 | 优先级 | 理由 |
|---|---|---|
| **Web管理后台** | P0 | 核心管理端，功能最全 |
| **PDA移动作业端** | P0 | 现场作业必需，直接影响效率 |
| **开放平台API** | P0 | 对接电商平台，实现自动化 |
| **客户门户Web端** | P1 | 提升客户体验，减少人工客服 |
| **数据看板/BI端** | P2 | 管理决策支持，可后期迭代 |
| **退货处理Web端** | P2 | 可先在Web管理后台实现，后期独立 |

## 7. 技术实现建议

### 7.1 技术栈推荐

| 端 | 技术栈 | 理由 |
|---|---|---|
| Web管理后台 | React + Ant Design | 组件化、企业级UI |
| PDA移动作业端 | React Native / Flutter | 跨平台、PDA兼容性好 |
| 客户门户Web端 | React + Material-UI | 轻量级、快速开发 |
| 开放平台API | Node.js / Java (Spring Boot) | 高并发、稳定 |
| 数据看板/BI端 | React + ECharts / D3.js | 可视化强 |
| 退货处理Web端 | React + Ant Design | 复用组件、快速开发 |

### 7.2 数据库设计要点

- **多语言支持**：所有用户可见字段需支持多语言存储
- **多币种支持**：金额字段需存储币种与汇率
- **审计日志**：所有操作需记录日志（谁、何时、做了什么）
- **数据隔离**：客户数据需逻辑隔离（按client_id分区）
