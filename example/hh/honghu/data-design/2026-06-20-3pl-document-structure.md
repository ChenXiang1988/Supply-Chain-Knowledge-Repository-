# 3PL 平台单据结构

文档类型：数据设计  
版本：V0.1  
日期：2026-06-20  
作者：Martin  
相关方：产品、架构、研发

## 1. 文档定位

当前 7 个场景文档中频繁出现"预约单""收货单""拣货任务"等单据名称，但没有一份文档定义它们是什么、有什么状态、之间如何关联。本文档补上这个缺口，作为场景文档的数据层面注解。

## 2. 单据总览

所有单据按两条链路、两层结构组织：**指令单据**（上层业务指令）和**作业单据**（现场执行记录）。入库、出库、盘点、售后四条链路结构一致。

```
                指令单据                    作业单据

S-03            入库订单 ──────┬──────→ 到货签收单
                              ├──────→ 验货入库单（含差异）
                              ├──────→ 楼层转运单（楼栋仓）
                              └──────→ 上架单
                                                │
                                                ▼
S-06            出库订单 ──────┬──────→ 拣货单（父子）
                              ├──────→ 补货单
                              ├──────→ 复核单
                              ├──────→ 打包单
                              └──────→ 发运单
                                                │
                                                ▼
                                              S-05
                                              库存流水
                                              库存快照

S-12            盘点指令 ──────────────→ 盘点单
（经营层）                                │
                                          ▼
                                      库存流水（盘点调整）

S-08            退货单(RMA) ───┬──────→ 退货到货签收单
（售后）                       ├──────→ 退货质检单
                               └──────→ 退货入库单
```

## 3. 单据定义

### 3.1 基础数据（非单据）

基础数据不是单据，不参与流转，仅作为业务引用的前提。

| 数据 | 说明 | 所属场景 |
|---|---|---|
| 客户组织 | 客户档案、账号、仓库绑定 | S-01 |
| 仓库配置 | 楼层、库区、库位 | S-02 |
| SKU 主数据 | 商品编码、条码、有效期声明、贴码规则 | S-02 |
| 客户服务规则 | 入库规则、库存分配策略、出库规则 | S-02 |

### 3.2 入库订单（Inbound Order）— 指令单据 — S-03

入库链的唯一指令单据。客户提交、仓库确认后，驱动后续所有作业单据。

| 字段 | 说明 | 来源 |
|---|---|---|
| inbound_order_id | 唯一编号，格式 IB-{日期}-{序号} | 系统生成 |
| org_id | 归属客户 | 客户登录上下文 |
| warehouse_id | 目标仓库 | 客户选择 |
| expected_date | 预计到仓日期 | 客户填写 |
| expected_time_slot | 时段（上午/下午/晚上） | 客户填写 |
| transport_type | 整车/零担/快递/自送 | 客户选择 |
| vehicle_plate | 车牌号 | 客户填写（可选） |
| contact_name/phone | 司机/联系人 | 客户填写（可选） |
| line_items | SKU 清单（SKU编码+预报数量+箱数） | 客户填写/模板导入 |
| status | 待确认 → 已确认 → 作业中 → 已完成 | 系统管理 |
| created_by | 客户管理员 ID | 登录上下文 |

### 3.3 到货签收单（Arrival Sign-off）— 作业单据 — S-03/S-04

车辆实际到达仓库时，收货员确认签收。从"预约"推进到"实际到货"的分界点。

| 字段 | 说明 | 来源 |
|---|---|---|
| signoff_id | 唯一编号，格式 AS-{日期}-{序号} | 系统生成 |
| inbound_order_id | 关联入库订单 | 到货登记时关联 |
| warehouse_id | 仓库 | 继承 |
| floor | 收货所在楼层 | 收货员 Handheld 上下文 |
| actual_cases | 实际到货箱数 | 收货员清点 |
| actual_pallets | 实际到货托盘数 | 收货员清点 |
| outer_condition | 外包装状态（正常/破损/变形） | 收货员目测 |
| signed_off_by | 收货员 ID | 登录上下文 |
| signed_off_at | 签收时间 | 系统记录 |
| status | 已签收（单状态单据，签收即终态） | 系统管理 |

### 3.4 验货入库单（Inspection & Receipt）— 作业单据 — S-04

验货员逐箱核对后生成。差异直接记在本单内，不独立成单。

| 字段 | 说明 | 来源 |
|---|---|---|
| inspection_id | 唯一编号，格式 IR-{日期}-{序号} | 系统生成 |
| inbound_order_id | 关联入库订单 | 自动关联 |
| signoff_id | 关联到货签收单 | 自动关联 |
| line_items | 逐行验货结果 | 验货员扫描录入 |

**line_items 行项结构：**

| 字段 | 说明 |
|---|---|
| sku_id | SKU |
| expected_qty | 预报数量（来自入库订单） |
| actual_qty | 实收数量 |
| discrepancy_type | 无差异 / 少收 / 多收 / 破损 / 错货 |
| discrepancy_qty | 差异数量（无差异时=0） |
| photo_urls | 破损拍照（破损时必填） |
| resolution | 入库 / 隔离 / 拒收（有差异时班组长确认） |
| production_date | 生产日期（有保质期的SKU必录） |
| expiry_date | 到期日（系统 = 生产日期 + SKU保质期，可手动修正） |

**单头字段：**

| 字段 | 说明 |
|---|---|
| inspected_by | 验货员 ID |
| inspected_at | 验货完成时间 |
| reviewed_by | 班组长 ID（有差异时） |
| reviewed_at | 审批时间 |
| status | 验货中 → 待审批 → 已完成 |

### 3.5 楼层转运单（Floor Transfer）— 作业单据 — S-04

楼栋仓专有。平层仓不产生此单据。叉车工持此单执行跨层转运。

| 字段 | 说明 | 来源 |
|---|---|---|
| transfer_id | 唯一编号，格式 FT-{日期}-{序号} | 系统生成 |
| inbound_order_id | 关联入库订单 | 自动关联 |
| from_floor | 起始楼层 | 收货楼层（通常1F） |
| to_floor | 目标楼层 | 上架推荐库位的楼层 |
| items | 需转运的 SKU + 数量 | 验货完成后提取 |
| assigned_to | 叉车工 ID | 领取时填充 |
| completed_at | 到位确认时间 | 叉车工扫描目标楼层 |
| status | 待领取 → 转运中 → 已到位（单状态单据） | 系统管理 |

### 3.6 上架单（Putaway）— 作业单据 — S-04

上架员扫描确认商品已放入目标库位。上架完成即触发库存入账。

| 字段 | 说明 | 来源 |
|---|---|---|
| putaway_id | 唯一编号，格式 PA-{日期}-{序号} | 系统生成 |
| inbound_order_id | 关联入库订单 | 自动关联 |
| line_items | 逐行上架结果 | 上架员扫描录入 |

**line_items 行项结构：**

| 字段 | 说明 |
|---|---|
| sku_id | SKU |
| qty | 上架数量 |
| recommended_location | 系统推荐库位（含楼层） |
| actual_location | 实际库位（上架员扫描确认） |

**单头字段：**

| 字段 | 说明 |
|---|---|
| putaway_by | 上架员 ID |
| putaway_at | 上架完成时间 |
| status | 已上架（单状态单据） |

**上架完成后的系统动作：** 写入库存流水（change_type=上架入库）→ 更新库存快照（available_qty +N）→ 入库订单状态推进。

### 3.7 出库订单（Outbound Order）— 指令单据 — S-06— S-06

| 字段 | 说明 | 来源 |
|---|---|---|
| order_id | 系统唯一编号 | 系统生成 |
| org_id | 归属客户 | 客户登录上下文 |
| customer_ref | 客户订单号 | 客户填写 |
| order_type | B2B / B2C | 客户选择/系统判断 |
| warehouse_id | 出库仓库 | 客户选择 |
| receiver_name | 收货方/收件人 | 客户填写 |
| receiver_address | 收货地址 | 客户填写 |
| receiver_phone | 收货电话 | 客户填写 |
| line_items | 出库明细（SKU+数量+单价） | 客户填写 |
| cut_off_time | 截单时间 | 继承自客户服务规则 S-02 |
| status | 待校验 → 已承诺 → 拣货中 → 已发运 → 已取消 | 系统管理 |

### 3.8 拣货单（Picking Task）— S-07

| 字段 | 说明 | 来源 |
|---|---|---|
| pick_task_id | 唯一编号 | 系统生成 |
| order_id | 关联出库订单 | 订单承诺后自动生成 |
| parent_task_id | 父任务 ID | 班组长拆分子任务时填充（根任务=null） |
| floor | 本子任务所在楼层 | 拆分时确定 |
| sku_list | 本子任务的 SKU + 需求数量 | 拆分结果 |
| recommended_locations | 推荐库位（按此顺序拣） | 系统推荐 |
| assigned_to | 拣货员 ID | 班组长分派 / 拣货员领取 |
| status | 待领取 → 拣货中 → 已完成 | 系统管理 |

**拆分示例：**
```
父任务 PICK-0042 (订单 O-0042, 12个SKU, 跨3F/6F)
  ├── 子任务 PICK-0042-3F (3F拣货区, 4个SKU) → 张三
  └── 子任务 PICK-0042-6F (6F存储区, 8个SKU) → 李四
```

### 3.9 补货单（Replenishment Task）— S-07

| 字段 | 说明 | 来源 |
|---|---|---|
| replen_id | 唯一编号 | 系统生成 |
| pick_task_id | 关联拣货子任务 | 系统自动关联 |
| sku_id | 需补货的 SKU | 系统检测 |
| qty | 补货数量 | 系统计算 |
| from_location | 来源库位（存储区） | 系统查找 |
| to_zone | 目标区域（拣货区） | 系统确定 |
| assigned_to | 叉车工 ID | 叉车工领取 |
| status | 待领取 → 补货中 → 已到位 | 系统管理 |

### 3.10 复核单（Checking Task）— S-07

| 字段 | 说明 | 来源 |
|---|---|---|
| check_id | 唯一编号 | 系统生成 |
| order_id | 关联出库订单 | 所有子任务完成后自动生成 |
| expected_items | 应发清单（SKU+数量） | 订单 line_items |
| picked_items | 实拣清单（含短拣标记） | 拣货子任务汇总 |
| discrepancies | 复核差异（多货/错货/短少） | 复核员逐件扫描结果 |
| checked_by | 复核员 ID | 登录上下文 |
| status | 待复核 → 复核中 → 通过/不通过 | 系统管理 |

### 3.11 打包单（Packing Task）— S-07

| 字段 | 说明 | 来源 |
|---|---|---|
| pack_id | 唯一编号 | 系统生成 |
| order_id | 关联出库订单 | 复核通过后生成 |
| pack_details | 箱号/托盘号 + 内含 SKU | 打包员录入 |
| label_url | 标签/面单打印链接 | 系统生成 |
| packed_by | 打包员 ID | 登录上下文 |
| status | 待打包 → 打包中 → 已完成 | 系统管理 |

### 3.12 发运单（Shipment）— S-07

| 字段 | 说明 | 来源 |
|---|---|---|
| shipment_id | 唯一编号 | 系统生成 |
| order_id | 关联出库订单 | 打包完成后生成 |
| ship_method | 客户自提 / 委托发货 | 发运员选择 |
| shipped_by | 发运员 ID | 登录上下文 |
| shipped_at | 发运时间 | 系统记录 |
| status | 待发运 → 已发运 | 系统管理 |

### 3.13 库存流水（Inventory Transaction）— S-04 / S-07

| 字段 | 说明 | 来源 |
|---|---|---|
| txn_id | 唯一编号 | 系统生成 |
| sku_id | SKU | 关联操作 |
| warehouse_id | 仓库 | 关联操作 |
| location_id | 库位 | 关联操作 |
| change_type | 上架入库 / 订单占用 / 发运出库 / 取消占用 / 盘点调整 / 补货移库 | 系统判断 |
| change_qty | +/- 数量 | 操作录入 |
| related_doc_type | 关联单据类型 | 系统记录 |
| related_doc_id | 关联单据 ID | 系统记录 |
| created_at | 时间 | 系统记录 |

### 3.14 库存快照（Inventory Snapshot）— S-05

| 字段 | 说明 | 来源 |
|---|---|---|
| sku_id + warehouse_id | 聚合维度 | — |
| available_qty | 可用库存 | 流水聚合 |
| reserved_qty | 已占用（订单承诺后） | 流水聚合 |
| in_transit_qty | 在途（已预约未上架） | 流水聚合 |
| earliest_expiry | 最早到期日 | SKU + 生产日期计算 |
| updated_at | 最后更新时间 | 系统记录 |

### 3.15 盘点指令（Cycle Count Order）— 指令单据 — S-12

库存管理员或仓库经理发起盘点。指令指定盘点范围（按库区/按SKU/全仓）。

| 字段 | 说明 | 来源 |
|---|---|---|
| count_order_id | 唯一编号，格式 CC-{日期}-{序号} | 系统生成 |
| warehouse_id | 仓库 | 发起人选择 |
| scope_type | 按库区 / 按SKU / 按楼层 / 全仓 | 发起人选择 |
| scope_detail | 具体范围（库区编号 / SKU编码列表） | 发起人填写 |
| initiated_by | 发起人 ID | 登录上下文 |
| status | 待执行 → 执行中 → 已完成 | 系统管理 |

### 3.16 盘点单（Cycle Count Sheet）— 作业单据 — S-12

库存管理员持此单逐库位核对实物与系统库存。

| 字段 | 说明 | 来源 |
|---|---|---|
| count_sheet_id | 唯一编号 | 系统生成 |
| count_order_id | 关联盘点指令 | 自动关联 |
| line_items | 逐库位盘点结果 | 库存管理员录入 |

**line_items 行项结构：**

| 字段 | 说明 |
|---|---|
| location_id | 库位 |
| sku_id | SKU |
| system_qty | 系统库存数 |
| actual_qty | 实物盘点数 |
| diff_qty | 差异（实际 - 系统） |
| remark | 差异原因备注 |

**单头字段：**

| 字段 | 说明 |
|---|---|
| counted_by | 库存管理员 ID |
| counted_at | 盘点完成时间 |
| status | 已盘点 |

**盘点完成后的系统动作：** 差异行项写入库存流水（change_type=盘点调整）→ 更新库存快照。

### 3.17 退货单 RMA（Return Merchandise Authorization）— 指令单据 — S-08

消费者向货主发起退货，货主审核后生成 RMA 退货单。消费者不直接接触 3PL 平台。

| 字段 | 说明 | 来源 |
|---|---|---|
| rma_id | 唯一编号，格式 RMA-{日期}-{序号} | 系统生成 |
| org_id | 归属货主 | 货主操作上下文 |
| consumer_name | 消费者姓名 | 货主填写 |
| consumer_phone | 消费者电话 | 货主填写 |
| return_reason | 退货原因（质量问题/无理由/错发/其他） | 货主选择 |
| original_order_id | 原出库订单编号 | 货主关联 |
| expected_sku | 预计退回的 SKU + 数量 | 货主填写 |
| return_tracking | 消费者寄回快递单号 | 货主填写（可选） |
| status | 待审核 → 已批准 → 待收货 → 已收货 → 已完成 | 系统管理 |

### 3.18 退货到货签收单（Return Arrival Sign-off）— 作业单据 — S-08

消费者寄回的商品到达仓库时，收货员签收确认。

| 字段 | 说明 | 来源 |
|---|---|---|
| return_signoff_id | 唯一编号 | 系统生成 |
| rma_id | 关联 RMA 退货单 | 自动关联 |
| actual_cases | 实际到货件数 | 收货员清点 |
| outer_condition | 外包装状态 | 收货员目测 |
| signed_off_by | 收货员 ID | 登录上下文 |
| status | 已签收 |

### 3.19 退货质检单（Return Inspection）— 作业单据 — S-08

验货员对退回商品做质量判定：可再售、返修、或报废。

| 字段 | 说明 | 来源 |
|---|---|---|
| return_inspection_id | 唯一编号 | 系统生成 |
| rma_id | 关联 RMA 退货单 | 自动关联 |
| line_items | 逐行质检结果 | 验货员录入 |

**line_items 行项结构：**

| 字段 | 说明 |
|---|---|
| sku_id | SKU |
| actual_qty | 实际收到数量 |
| quality_result | 可再售 / 需返修 / 已报废 |
| photo_urls | 拍照（破损/质量问题） |
| remark | 备注 |

**单头字段：**

| 字段 | 说明 |
|---|---|
| inspected_by | 验货员 ID |
| reviewed_by | 班组长 ID（质检有争议时） |
| status | 质检中 → 已完成 |

### 3.20 退货入库单（Return Putaway）— 作业单据 — S-08

质检判定后，可再售商品重新上架入库。不可再售的走返修或报废流程（线下）。

| 字段 | 说明 | 来源 |
|---|---|---|
| return_putaway_id | 唯一编号 | 系统生成 |
| rma_id | 关联 RMA 退货单 | 自动关联 |
| line_items | 逐行上架结果 | 上架员录入 |

**line_items 行项结构：**

| 字段 | 说明 |
|---|---|
| sku_id | SKU |
| qty | 入库数量（仅"可再售"的 SKU） |
| location_id | 实际库位 |

**单头字段：**

| 字段 | 说明 |
|---|---|
| putaway_by | 上架员 ID |
| status | 已入库 |

**入库后的系统动作：** 写入库存流水（change_type=退货入库）→ 更新库存快照 → 通知货主 RMA 已完成。```
入库订单(指令) ──1:1──→ 到货签收单
              ├──1:1──→ 验货入库单（含差异）
              ├──0:N──→ 楼层转运单（楼栋仓）
              └──1:1──→ 上架单
                            │
                            ▼
                        库存流水 ──→ 库存快照

出库订单(指令) ──1:N──→ 拣货任务(父) ──1:N──→ 拣货任务(子)
              │
              ├──0:N──→ 补货任务
              │
              └──1:1──→ 复核任务 ──→ 打包任务 ──→ 发运单
                                                     │
                                                     ▼
                                                 库存流水 ──→ 库存快照
```

## 5. 与场景的对应

| 单据 | 类型 | 创建于 | 消费于 |
|---|---|---|---|
