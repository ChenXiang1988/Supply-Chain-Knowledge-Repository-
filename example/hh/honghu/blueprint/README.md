## 📌 文档定位

### 战略规划类
- **全球3PL平台蓝图**: 总纲文件，定义平台目标、设计原则、总体架构
- **蓝图实现路径**: 可执行的实施路线，明确先做什么、先验证什么
- **泓湖试点范围**: 冻结第一期范围，明确仓型、客户类型、场景取舍

### 业务设计类
- **业务能力树**: 平台能力分层，明确基础底座、业务层能力、区域差异能力
- **领域模型**: 核心对象、关系、聚合边界、状态机和领域事件
- **国家/仓型配置**: 新国家和新仓型上线配置清单，支持快速复制
- **行业参考案例**: 可直接借鉴的行业样本

### 权限与流程类
- **角色权限矩阵**: 内部团队、客户、合作方的权限边界和数据范围
- **主流程泳道图**: 订单到履约到结算的主链路（旧版）
- **试点主干流程V2**: 试点范围的端到端流程（当前有效）

## 🎯 使用指南

**不同角色如何使用这些文档：**

- **项目经理**: 重点关注"战略规划类"文档，了解项目范围和路径
- **产品经理**: 重点关注"业务设计类"和"权限与流程类"文档，了解功能设计
- **架构师**: 重点关注"业务设计类"文档，特别是领域模型和能力树
- **研发**: 重点关注"权限与流程类"文档，特别是流程设计文档
- **测试**: 参考所有文档，特别是流程设计，编写测试用例
- **实施**: 重点关注"战略规划类"和"国家/仓型配置"文档
