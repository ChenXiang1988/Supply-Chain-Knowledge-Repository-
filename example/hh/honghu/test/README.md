## 📝 测试文档规范

### 测试用例结构

每个测试用例建议包含以下字段：

| 字段 | 说明 |
|------|------|
| 用例编号 | 唯一标识，如 TC-S01-001 |
| 用例标题 | 简要描述测试目的 |
| 前置条件 | 执行测试所需的条件 |
| 测试步骤 | 详细的测试步骤 |
| 预期结果 | 期望的测试结果 |
| 实际结果 | 实际的测试结果 |
| 测试状态 | 通过/失败/阻塞 |
| 备注 | 其他说明 |

### 命名规范

- **测试计划**: `test-plan-<模块>-<版本>.md`
- **测试用例**: `test-case-<场景>-<编号>.md`
- **测试报告**: `test-report-<版本>-<轮次>.md`
