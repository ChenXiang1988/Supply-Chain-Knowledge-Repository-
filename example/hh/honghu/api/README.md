## 📝 API文档规范

### 文档结构

每个API文档建议包含以下章节：

1. **接口概述**: 简要描述接口用途
2. **请求URL**: 接口的URL路径
3. **请求方法**: GET、POST、PUT、DELETE等
4. **请求参数**: 详细的参数说明（参数名、类型、是否必填、说明）
5. **响应参数**: 详细的响应说明
6. **请求示例**: 实际的请求示例
7. **响应示例**: 实际的响应示例
8. **错误码**: 可能的错误码及说明
9. **注意事项**: 特殊说明

### 命名规范

- **文件名**: `api-<模块>-<接口名>.md`
- **示例**: `api-order-sync.md`, `api-inventory-query.md`

### 工具推荐

- **API设计**: 使用Swagger/OpenAPI规范设计API
- **文档生成**: 使用Swagger UI或类似工具生成可视化文档
- **接口测试**: 使用Postman或类似工具测试接口

## 📝 维护规则

1. **新增API**: 新增API文档时，请更新本文档的"文档规划"部分
2. **文档更新**: API变更时，请及时更新对应文档
3. **版本管理**: API有重要变更时，建议在文档中记录版本变更历史
4. **示例优先**: 提供清晰的请求/响应示例，便于理解
